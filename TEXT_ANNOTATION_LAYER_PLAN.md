# План: окремий декоративний шар текстів

## Мета

Відв'язати переміщення/редагування підписів від `calc_results.sections`, щоб:

- підписи не ламались при частковому перерахунку блоків;
- не було колізій ключів між різними сабмейнами;
- можна було редагувати позицію/кут/видимість тексту без повторного розрахунку.

## Мінімальний MVP (без великого рефакторингу)

1. Додати в стан проєкту колекцію `annotations`:
   - `id` (рядок, стабільний),
   - `type` (`submain_section`, `valve`, `custom`),
   - `anchor_ref` (прив'язка до об'єкта: `block_idx`, `sm_idx`, `section_index`, `sub_idx`),
   - `text`,
   - `pos` (`x`, `y`),
   - `angle_deg` (опційно),
   - `style` (`color`, `font_size`, `visible`),
   - `locked` (bool).

2. У `redraw()` малювати підписи через шар `annotations`:
   - спочатку геометрія (лінії),
   - потім підписи з шару.

3. Для сумісності:
   - якщо `annotations` порожній, згенерувати runtime-чернетку з поточної логіки `section_label_pos`;
   - запис у JSON робити в нове поле `annotations`.

4. Режим редагування підписів:
   - 1-й ЛКМ: вибір анотації;
   - рух миші: preview;
   - 2-й ЛКМ: commit `pos`;
   - `Esc`: cancel.

5. Інструменти MVP:
   - перемістити підпис;
   - скинути позицію до auto;
   - приховати/показати підпис.

## Дані в JSON

Пропозиція:

```json
{
  "annotations": [
    {
      "id": "sm:3:sec:5:sub:0",
      "type": "submain_section",
      "anchor_ref": {"block_idx": 1, "sm_idx": 3, "section_index": 5, "sub_idx": 0},
      "text": "PVC d50/6 L=34.2m",
      "pos": {"x": 123.45, "y": -67.89},
      "angle_deg": 12.0,
      "style": {"color": "#000000", "font_size": 10, "visible": true},
      "locked": false
    }
  ]
}
```

## Кроки впровадження (поетапно)

### Етап 1: інфраструктура
- Додати поле `self.annotations` + серіалізацію в `file_io_impl.py`.
- Додати генератор ключів `annotation_id`.
- Додати runtime-міграцію зі старого `section_label_pos`.

### Етап 2: рендер
- Винести малювання тексту секцій у функцію `draw_annotations()`.
- Використовувати `annotations` як єдине джерело позиції тексту.

### Етап 3: взаємодія
- Режим `SUB_LABEL` переключити на роботу з `annotations`.
- Додати `Esc` для відміни drag.
- Додати tooltip/статус для активного редагування.

### Етап 4: UX
- Контекстне меню: reset/hide/lock.
- Кнопка "Скинути підписи блоку до auto".

## Ризики

- Потрібно акуратно зберегти зворотну сумісність зі старими JSON.
- Важливо уникнути дублювання (коли одночасно є старі `section_label_pos` і нові `annotations`).

## Критерії готовності

- Підписи рухаються і не залежать від того, перерахований блок чи ні.
- Часткове редагування сабмейну не створює фантомних підписів.
- Після збереження/завантаження позиції та стиль підписів відновлюються 1:1.
