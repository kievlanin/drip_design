\# Алгоритм розв'язання прямої задачі (Shooting Method)



Оскільки витрата некомпенсованої латералі залежить від тиску, який сам залежить від витрати, ми використовуємо ітераційний підбір.



\### Вхідні дані:

\- `H\_target`: Відомий тиск у точці врізки (з сабмейну).

\- `Tolerance`: Допустима похибка (наприклад, 0.01 м).



\### Алгоритм реалізації:



1\. \*\*Визначення функції помилки:\*\*

&#x20;  Створіть функцію `f(H\_guess)`, яка:

&#x20;  - Приймає припущення про тиск на кінці латералі.

&#x20;  - Викликає `backwards\_step\_method(H\_guess)`.

&#x20;  - Повертає різницю: $Error = H\_{calculated\\\_start} - H\_{target}$.



2\. \*\*Метод пошуку кореня (на вибір):\*\*



&#x20;  \*\*А. Метод бісекції (найбільш стабільний):\*\*

&#x20;  - Визначити межі: `H\_min = 0`, `H\_max = H\_target`.

&#x20;  - Поки `(H\_max - H\_min) > Tolerance`:

&#x20;    - `H\_mid = (H\_min + H\_max) / 2`

&#x20;    - Якщо `f(H\_mid) > 0`, то `H\_max = H\_mid`

&#x20;    - Інакше `H\_min = H\_mid`



&#x20;  \*\*Б. Метод Ньютона (найшвидший):\*\*

&#x20;  - Використовуйте, якщо потрібна висока швидкість розрахунку великих масивів даних. Потребує обчислення похідної $f'(H)$.



3\. \*\*Фіналізація:\*\*

&#x20;  - Після знаходження оптимального $H\_{end}$, виконайте фінальний прохід "Backwards Step", щоб отримати:

&#x20;    - `Q\_total` (загальна витрата латералі).

&#x20;    - Масив локальних витрат кожної крапельниці для карти поливу.

Для реалізації розрахунку можна використати бібліотеку scipy.optimize
from scipy.optimize import bisect
